Pipeline per install email · reference only
Pipeline per install email
  1. Parse the email — grab SO number, ticket ID, driver name, serial numbers, driver comments, photos.
  2. Upload photos → HubSpot ticket (associated ticket, not the deal).
  3. Populate serial number + installer fields on the ticket.
  4. Post a note tagging the deal owner. If driver comments are material (damage, access issue, customer complaint), tag Courtney too.
  5. Email the deal owner a short wrap-up in your voice (install complete, any flags).
  6. Stage move: Install Scheduled → Install Complete.
  7. Fulfil the SO in Inflow — deduct stock, close the picking list.
Risks / edge cases
  • Stage guard — skill only processes tickets in Install Scheduled. If Courtney emails a report for a ticket in the wrong stage, it skips silently. Spot-check the Slack summary for misses.
  • Driver comments — don't skim these. "Install complete" + "customer wasn't happy with placement" = a sales problem, not an ops close.
  • Duplicate reports — 48h dedupe handles same-day re-sends, but if Courtney forwards the same report 3 days later, you'll get a second note. Watch for it.
  • Serial number mismatches — if the serial on the photo doesn't match the SO line item, that's an inventory/picking error upstream. Flag Courtney.
